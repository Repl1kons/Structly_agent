# Structly Agent

Локальный агент Structly запускается рядом с базой данных клиента и
выполняет задачи, получаемые от backend.

## Что делает агент

Агент работает в цикле и:

- проверяет подключение к базе (`SELECT 1`)
- выполняет `pg_dump --schema-only` (только схема, без данных)
- отправляет результат обратно в Structly backend

## Безопасность

Агент спроектирован с учетом безопасной работы с секретами:

- При первом запуске агент **генерирует пару ключей (RSA)**:

    - приватный ключ хранится локально
    - публичный ключ регистрируется на backend

- Пароль к PostgreSQL передается в виде:

    - зашифрованном публичным ключом агента (`RSA-OAEP + SHA-256`)
    - расшифровывается **только локально** приватным ключом агента

- Весь обмен с backend происходит по **HTTPS (TLS)**

- Подключение к PostgreSQL:

    - поддерживает SSL (`ssl_on = true`)
    - может быть отключено (`ssl_on = false`) --- зависит от
      конфигурации

⚠️ Важно: - приватный ключ хранится локально и должен быть защищен на
уровне инфраструктуры (container / VM) - шифруются секреты и сетевые
соединения, но уровень защиты подключения к БД зависит от настроек SSL

## Переменные окружения

| Переменная                     | Описание                      | По умолчанию                   |
|--------------------------------|-------------------------------|--------------------------------|
| `STRUCTLY_BACKEND_URL`         | URL backend сервера Structly  | `https://structly.elevo.space` |
| `STRUCTLY_AGENT_TOKEN`         | Токен агента                  | —                              |
| `STRUCTLY_AGENT_POLL_INTERVAL` | Интервал опроса backend (сек) | `10`                           |
| `STRUCTLY_AGENT_TIMEOUT`       | HTTP timeout (сек)            | `120`                          |
| `PG_DUMP_BIN`                  | Путь к `pg_dump`              | `pg_dump`                      |
| `PSQL_BIN`                     | Путь к `psql`                 | `psql`                         |
| `STRUCTLY_AGENT_STATE_DIR`     | Директория для ключей агента  | `/agent/state`                 |

## Сборка

``` bash
docker build -t structly-agent .
```

## Запуск

``` bash
docker run --rm -e STRUCTLY_AGENT_TOKEN={YOUR_AGENT_TOKEN} structly-agent
```

## Как это работает (кратко)

1. Агент стартует и регистрирует свой публичный ключ
2. Backend выдает задания (jobs)
3. Агент:
    - забирает задание
    - подключается к БД
    - выполняет dump схемы
4. Результат отправляется обратно в backend
